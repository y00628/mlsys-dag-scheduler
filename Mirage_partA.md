Mirage 實作指南：Part A - IR / Graph Representation此文件定義了 Mirage 超級最佳化器中的核心資料結構與圖形表示法（$\mu$Graphs），包含 Kernel、Thread Block 與 Thread 三個層級的實作細節。1. 多層級圖形架構 (Multi-Level Graph Hierarchy)Mirage 使用層次化的圖結構來描述 GPU 計算：1.1 KNGraph (Kernel Graph) - 核心層級 Nodes (KNNode): 代表在整個 GPU 上運行的一個 Kernel。可以是預定義運算（如 cuBLAS 的 MatMul）或由 TBGraph 定義的「自訂 Kernel」 。Edges (KNEdge): 代表存在於 GPU Device Memory 中的張量（DTensor） 。不同 Kernel 之間只能透過 Device Memory 共享數據 。1.2 TBGraph (Threadblock Graph) - 執行緒區塊層級 Nodes (TBNode): 代表 Thread Block 內部的細粒度運算（Block Operators），如 Input Iterator、MatMul、Accumulator 等 。Edges (TBEdge): 代表存在於 GPU Shared Memory 中的張量（STensor） 。記憶體策略: 所有中間張量必須儲存在 Shared Memory 以提高頻寬；若尺寸超過限制，則需將計算拆分到多個 TBGraph 。1.3 Thread Graph - 執行緒層級 功能: 將計算範圍縮小到單一執行緒，負責將數據從 Shared Memory 載入到 Register File，進行運算後存回 。2. 張量元數據與存取映射 (Tensor Metadata & Mappings)2.1 張量屬性 DTensor: 屬性包含 shape, dtype, layout (Linearized order in memory) 。STensor: 除了基礎屬性，還需考慮 Shared Memory 的 swizzle 模式以避免 Bank Conflict。2.2 關鍵存取映射 (Indexing Mappings)這是實作 TBGraph 與 KNGraph 銜接的核心邏輯：imap (Input Map): 定義如何將 KNGraph 的輸入張量分割給各個 Thread Block 。映射 Grid 軸 (x, y, z) 到數據維度或複寫維度 (Replica) 。omap (Output Map): 定義各個 Block 的輸出如何拼接回最終的輸出張量 。Grid 軸必須映射到不重疊的數據維度 。fmap (For-loop Map): 定義在 For-loop 迭代中，Input Iterator 應該讀取張量的哪個部分 。3. 運算子類型系統 (Operator System)實作中需區分以下類型 ：類型運算子範例支援層級載入/儲存InIter, OutSaver, InputIterator, OutputSaverB, T核心計算Matmul, Conv, EwAdd, EwMul, EwDiv, EwExpK, B, T輔助/變換Reshape, Sqrt, SiLU, Repeat, AccumK, B自訂GraphDefinedOperator (指向下層 TBGraph)K4. 圖合法性驗證 (Graph Validity)在 Copilot 生成圖形後，必須通過以下檢查才算合法 ：規格匹配: 所有 Operator 的輸入/輸出 Tensor Shape 與屬性必須相符 。記憶體約束:KNGraph 的張量需 fit 入 Device Memory 。TBGraph 的張量需 fit 入 Shared Memory 。Thread Graph 的張量需 fit 入 Register File 。結構約束 (For-loop Body): 在含有 For-loop 的 TBGraph 中，任何路徑必須嚴格遵循：InputIterator -> Accumulator -> OutputSaver 。5. 實作任務清單 (Implementation Checklist)數據結構實作[ ] 實作 Tensor 類：包含 shape, layout, memory_type (Global/Shared/Register)。[ ] 實作 KNGraph：包含 nodes (Operator 列表) 與 edges (Tensor 連結)。[ ] 實作 TBGraph：繼承自 KNGraph 但增加 grid_dim, for_loop_dim 屬性。核心邏輯實作[ ] 實作 imap/omap/fmap 的解析器：處理如何根據 Block ID 與 Loop Index 計算 Tensor Offset。[ ] 實作 GraphDefinedOperator：允許 KNNode 封裝一個完整 TBGraph 的邏輯。[ ] 實作 check_validity() 函數：檢查所有運算子的 Shape 與 GPU 記憶體上限 。程式碼生成預備[ ] 實作一個遞歸函數，能將 KNGraph 展開為層次結構，並識別出哪些部分需要生成 Triton 或 CUDA PTX 代碼 。
